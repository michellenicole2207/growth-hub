# Chapter 3 - Static Testing Mind Map

```mermaid
mindmap
  root((Ch3 Static Testing))
    Static Testing Basics
      Without executing the software
      Manual reviews
      Automated static analysis tools
    Work Products Examined
      Requirements and user stories
      Source code
      Test plans and test cases
      Designs and models
      Project documentation
    Value of Static Testing
      Catches defects early
      Finds what dynamic testing cannot
        Unreachable code
        Non-executable documents
        Design flaws
      Builds shared understanding
      Cost-effective
    Static vs Dynamic Testing
      Static - no execution needed
      Dynamic - must run the software
      Static finds defects directly
      Dynamic reveals defects via failures
    Review Process - 5 Steps
      1 Planning
      2 Review initiation
      3 Individual review
      4 Communication and analysis
      5 Fixing and reporting
        Author fixes defects
        Review leader writes report
    Roles in a Review
      Manager
      Author
      Moderator
      Scribe - mandatory in walkthrough
      Reviewer
      Review leader
    Review Types
      Informal - no defined process
      Walkthrough - author-led
      Technical review - moderator or peer leader
      Inspection - most formal
        Author cannot be leader or scribe
    Success Factors
      Clear objectives
      Right review type chosen
      Small chunks reviewed
      Management support
      Adequate preparation time
```
