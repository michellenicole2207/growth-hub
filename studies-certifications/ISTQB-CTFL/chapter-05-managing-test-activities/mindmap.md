# Chapter 5 - Managing Test Activities Mind Map

mindmap
  root((Ch5 Managing Test Activities))

      Test plan content
        Context and scope

      Unit tests - many fast cheap
      Integration tests - some
      UI and E2E - few slow brittle
    Testing Quadrants
      Q1 Technology Supporting - unit tests
      Q2 Business Supporting - functional tests
      Q3 Business Critiquing - exploratory UAT
      Q4 Technology Critiquing - performance security
    Risk-Based Testing
      Product risk vs Project risk
      Risk identification
      Risk assessment
        Likelihood x Impact
    Test Monitoring and Control
      Metrics collected
        Tests executed passed failed
        Defect detection rate
        Coverage
      Control actions
        Reprioritize tests
        Adjust scope or schedule
      Progress reports
      Completion reports

        Expected vs actual result
